Practical 5
A) Monitor and administer undo 
Identify the Current Undo Tablespace:

SHOW PARAMETER undo_tablespace;

This shows which undo tablespace the instance is using

SQL> SELECT name, value FROM v$parameter WHERE name LIKE 'undo%";

SQL> SELECT
2 	a.tablespace_name,
3 	a.bytes / 1024 / 1024 AS size_mb,
4 	b.bytes / 1024 / 1024 AS used_mb,
5 	(a.bytes NVL(b.bytes,0)) / 1024 / 1024 AS free_mb
6 FROM
7 	dba_data_files a,
8 	(SELECT SUM(bytes) bytes, file_id
9 	FROM dba undo extents
10 	WHERE status = 'ACTIVE'
11 	GROUP BY file_id) b
12 WHERE a.file_id=b.file_id(+)
13 	AND a.tablespace_name IN (SELECT value FROM v$parameter WHERE name = 'undo_tablespace');

SQL> SELECT
	s.sid, 
	s.serial#, 
	t.used urec AS "Undo Records"
	t.used_ublk AS "Undo Blocks",
	s.username 
FROM 
	v$transaction t,
	v$session s
WHERE
	t.ses_addr s.saddr;

practical 6
Create Net Service Aliases
ORCLDB =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = orcl)
    )
  )
Test Connection Failover
tnsping ORCLDB

Enable Failover Settings

To enable load balancing and failover:
ORCL_FAILOVER =
  (DESCRIPTION =
    (LOAD_BALANCE = ON)
    (FAILOVER = ON)
    (ADDRESS_LIST =
      (ADDRESS = (PROTOCOL = TCP)(HOST = host1)(PORT = 1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = host2)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVICE_NAME = orcl)
    )
  )
Test Connection Failover

Stop one listener and connect again:
lsnrctl stop LISTENER
sqlplus system/password@ORCL_FAILOVER
It will automatically connect to the standby listener.
